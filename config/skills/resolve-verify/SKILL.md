# VERIFY -- Review QA Flows + Generate verify.sh

Review the diagnosis QA Reproduction Flows with the user and generate verify.sh.

**Input:** `$ARGUMENTS` = `{slug}`. Diagnosis at `.workflow/resolve/{slug}/diagnosis.md` with Status: DIAGNOSED.

## Boundaries

- **Authority:** ONLY set Status to `VERIFIED`. Never write INVESTIGATING, FIXING, CERTIFYING, or FAILED.
- **Write scope:** `.workflow/resolve/{slug}/diagnosis.md`, `.workflow/build/verify.sh`.

---

## Step 0: LOAD TOOLS

Ensure AskUserQuestion is available: `ToolSearch("select:AskUserQuestion", max_results: 1)`

---

## Step 1: Read Diagnosis

Read `.workflow/resolve/{slug}/diagnosis.md`. Understand the root cause, hotspots, and QA Reproduction Flows. If helpful, explore relevant code/pages to validate the flows make sense.

## Step 2: Present Diagnosis + Ask Approval

Build a preview with the diagnosis summary and QA flows:

```
Bug: {one-line summary}
Root Cause: {brief root cause}
Severity: {STANDARD | COMPLEX}
Suggested Fix: {brief approach}

QA Reproduction Flows:

R1: {flow name}
  - preconditions: {state needed}
  - human-steps:
    1. {step}
    2. {step}
  - expected-bug: {what the bug looks like}
  - expected-fixed: {what correct behavior looks like}
```

Call AskUserQuestion:
- Question: "Do you approve the diagnosis and QA reproduction flows?"
- Header: "Bug Diagnosis"
- Options: `Approve diagnosis & flows` / `Needs changes`
- Preview: the full diagnosis summary + QA flows above

### If "Approve diagnosis & flows"

1. Set Status -> `VERIFIED` in `.workflow/resolve/{slug}/diagnosis.md`.
2. Write `.workflow/build/verify.sh` with V1-V3 baselines **ONLY IF it does not already exist**. If it exists (from a prior /build), skip — the existing verify.sh already covers V1-V3 and may include --full support that must not be overwritten:

```bash
#!/bin/bash
# Auto-generated by /resolve -- matches diagnosis verification
set -euo pipefail
SLUG="${1:?Usage: verify.sh <slug>}"
FAILED=0

run_check() {
  local name="$1"; shift
  echo "=== $name ===" >&2
  local output
  if output=$("$@" 2>&1); then
    echo "  PASS" >&2
  else
    echo "  FAIL" >&2
    echo "$output" >&2
    FAILED=1
  fi
}

run_check "V1: Unit tests" npm test -- --run
run_check "V2: TypeScript" npx tsc --noEmit
run_check "V3: Build" npm run build

exit $FAILED
```

3. Return `{slug}: VERIFIED`

### If "Needs changes"

Read feedback, update QA Reproduction Flows (and Root Cause / Suggested Fix if warranted), re-ask. Max 3 loops.

## Output

Return ONLY: `{slug}: VERIFIED`
